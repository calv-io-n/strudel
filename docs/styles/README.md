# Styles

This folder is the *mood* layer of the docs.

- **Genres** (`../genres/`) are tempo + drum-grid recipes — DnB, trance, synthwave.
- **Styles** (here) are sonic-character recipes that cut across genres — melodic, dark, hypnotic, etc. You can apply any style to any genre.

A "melodic dark DnB" track is the genre file (`dnb.md`) for the skeleton, plus the style files (`melodic.md`, `dark.md`) for the character.

## Available styles

- [Melodic](melodic.md) — tunes, hooks, harmonic motion
- [Atmospheric](atmospheric.md) — pads, space, slow movement
- [Dark](dark.md) — minor modes, weight, dread
- [Uplifting](uplifting.md) — major modes, ascent, euphoria
- [Dreamy](dreamy.md) — chorused, detuned, reverb-soaked
- [Hypnotic](hypnotic.md) — locked repetition with slow drift
- [Glitchy](glitchy.md) — stutters, micro-edits, computational artifacts
- [Organic](organic.md) — acoustic, hand-played feeling
- [Aggressive](aggressive.md) — distortion, density, hard transients
- [Lo-fi](lofi.md) — warmth, hiss, bandwidth limitation

## How to use

Pick a genre file as your starting point. Then layer in 1-2 style techniques from the relevant style files. Don't try to combine everything at once — pick a primary style and a secondary style and let them dominate.

Each style file follows the same shape:
1. Definition (what is the *feeling*)
2. Sound palette
3. Key tricks (with code)
4. Starter pattern
5. Knobs to turn
6. Common mistakes
7. Combines well with

## Code accuracy note

These were written by translating general electronic-production knowledge into Strudel idioms, not by verifying against a running install. Treat the example code as a *sketch* of the technique — function names and exact syntax may need fixes when you run them. The *concepts* are sound; the *syntax* needs a pass through `scratch.strudel` before relying on it.
